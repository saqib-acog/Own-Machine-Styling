How to make use of this system

As a user:

  1. See if there is a container running on standard or specified port with the given name
  2. See if there is a specified website in the configuration

As a developer:

  1. Run a container as you would. If you are running in a standard port (80, 8000, 8888, 3000, 4000), it will be picked up.
  2. If you are running on a non standard port, use the label as --label port=yournonstandardport
  3. If you don't want to use the LDAP security, use the label as --label security=none
  4. An optional good practice is --label description="You can write about the container".
  5. You can ask the system to ignore this container by --label ignore
  6. The discovery happens dynamically. Just refresh this page!
  7. If you want to statically specify a service (useful to run outside the container), you can contact helpdesk.
Here is an example: docker run -d --rm -e PASSWORD=yourpassword --name ${USER}-rstudio --label description="RStudio service" rocker/rstudio

Best practices

  1. Use '--rm' so that it is deleted after usage.
  2. Use the convention '--name project-${USER}-servicename' for clear ownership of the running container. If no project, skip it.
